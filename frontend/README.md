# تراث المندي - نظام المحاسبة للمطاعم

نظام محاسبة متكامل للمطاعم مبني بتقنيات حديثة مع دعم كامل للغة العربية واتجاه RTL.

## التقنيات المستخدمة

- **React 19.2** - مكتبة واجهة المستخدم
- **Vite 7.2** - أداة البناء والتطوير
- **TypeScript 5.9.3** - لغة البرمجة
- **TailwindCSS 4.1.17** - إطار عمل التصميم مع دعم RTL
- **React Router 7.9.6** - التنقل بين الصفحات
- **Zustand 5.0.8** - إدارة حالة المصادقة
- **TanStack Query 5.90.9** - إدارة حالة الخادم
- **Axios** - طلبات HTTP

## البنية الهيكلية

```
src/
├── components/          # المكونات القابلة لإعادة الاستخدام
│   ├── common/         # المكونات المشتركة
│   └── layout/         # مكونات التخطيط
├── pages/              # صفحات التطبيق
│   ├── auth/          # صفحات المصادقة
│   └── dashboard/     # صفحات لوحة التحكم
├── hooks/              # React Hooks مخصصة
├── stores/             # مخازن Zustand
├── services/           # خدمات API
├── types/              # تعريفات TypeScript
├── utils/              # دوال مساعدة
├── App.tsx            # المكون الرئيسي
└── main.tsx           # نقطة الدخول
```

## الميزات

### دعم اللغة العربية (RTL)
- واجهة كاملة بالعربية
- دعم اتجاه RTL في جميع المكونات
- خطوط عربية محسنة (Cairo & Tajawal)
- TailwindCSS مع إضافة RTL

### المصادقة والأمان
- نظام مصادقة متكامل باستخدام JWT
- تحديث تلقائي للرموز
- حماية المسارات
- إدارة الجلسات

### إدارة الحالة
- **Zustand** لحالة المصادقة والحالة العامة
- **TanStack Query** لطلبات API والتخزين المؤقت
- استمرارية الحالة في localStorage

### طلبات API
- Axios مع Interceptors
- معالجة الأخطاء
- تحديث تلقائي للرموز
- إعدادات مهلة محسنة

## التثبيت

1. تثبيت المكتبات:
```bash
npm install
```

2. نسخ ملف البيئة:
```bash
cp .env.example .env
```

3. تعديل متغيرات البيئة في `.env`:
```env
VITE_API_URL=http://localhost:8000/api
```

## التشغيل

### وضع التطوير
```bash
npm run dev
```
سيعمل التطبيق على `http://localhost:3000`

### البناء للإنتاج
```bash
npm run build
```

### معاينة البناء
```bash
npm preview
```

### فحص الكود
```bash
npm run lint
```

## المسارات

- `/login` - صفحة تسجيل الدخول
- `/dashboard` - لوحة التحكم (محمية)
- `/sales` - إدارة المبيعات (قريباً)
- `/purchases` - إدارة المشتريات (قريباً)
- `/inventory` - إدارة المخزون (قريباً)
- `/reports` - التقارير (قريباً)
- `/settings` - الإعدادات (قريباً)

## أفضل الممارسات

### أسماء الملفات
- استخدم PascalCase للمكونات: `LoginPage.tsx`
- استخدم camelCase للملفات الأخرى: `authStore.ts`

### استيراد المسارات المختصرة
```typescript
import { useAuth } from '@hooks/useAuth';
import LoginPage from '@pages/auth/LoginPage';
import { User } from '@types/auth.types';
```

### معالجة الأخطاء
جميع طلبات API تتضمن معالجة أخطاء مناسبة ورسائل واضحة.

### TypeScript
استخدام أنواع صارمة في جميع أنحاء التطبيق.
